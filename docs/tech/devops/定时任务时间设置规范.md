## 一句话结论

- `fixedDelay`：上次执行结束后，等待固定时长再执行。
- `fixedRate`：按固定频率触发任务。
- `cron`：在明确时间点触发任务。

## 三种调度方式怎么选

| 方式 | 触发基准 | 适用场景 | 示例 |
| --- | --- | --- | --- |
| `fixedDelay` | 上次执行结束后 | 轮询、补偿、执行耗时不稳定任务 | `@Scheduled(fixedDelay = 60 * 1000L)` |
| `fixedRate` | 固定频率 | 统计、采集、缓存刷新 | `@Scheduled(fixedRate = 60 * 1000L)` |
| `cron` | 固定时间点 | 日结、报表、对账 | `@Scheduled(cron = "0 0 1 * * ?")` |

> 注意：Spring 默认调度通常是单线程。即使是 `fixedRate`，上次没执行完时也可能顺延。配置线程池后，才更容易出现并发执行，需要关注幂等和锁。

## Spring/Quartz Cron 格式

Spring 常用 6 位表达式：

```text
秒 分 时 日 月 周
```

示例：

```text
0 0 1 * * ?
```

含义：每天 `01:00:00` 执行。

关键点：

- `日` 和 `周` 通常一位写具体值，另一位写 `?`。
- Spring/Quartz 与 Linux `crontab` 不是同一种格式，不能直接互抄。

## 常用特殊字符

- `*`：任意值
- `?`：不指定（常用于日/周）
- `,`：多个值
- `-`：范围
- `/`：步长
- `L`：最后（如月末）
- `W`：工作日

## 常用模板（可直接改）

| 场景 | 表达式 |
| --- | --- |
| 每 5 分钟 | `0 */5 * * * ?` |
| 每小时整点 | `0 0 * * * ?` |
| 每天凌晨 1 点 | `0 0 1 * * ?` |
| 每天 9 点和 18 点 | `0 0 9,18 * * ?` |
| 工作日 10 点 | `0 0 10 ? * MON-FRI` |
| 每月最后一天 23 点 | `0 0 23 L * ?` |

## 和 Linux crontab 的区别

- Linux 常见 5 位：`分 时 日 月 周`
- Spring 常见 6 位：`秒 分 时 日 月 周`

同样表示“每天 1 点”：

- Linux：`0 1 * * *`
- Spring：`0 0 1 * * ?`

## 实战建议

1. 需要“间隔执行”优先选 `fixedDelay` / `fixedRate`。
2. 需要“固定时刻执行”优先选 `cron`。
3. 生产环境建议显式声明时区，避免服务器时区导致跑偏。

```java
// 每天凌晨 1 点执行
@Scheduled(cron = "0 0 1 * * ?", zone = "Asia/Shanghai")
public void dailyTask() {
}
```
