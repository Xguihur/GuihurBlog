## 场景

线上日志通常会同时存在两类文件：

- 普通文本日志，如 `sys-info.log`
- 压缩日志，如 `sys-info.2026-04-07.0.log.gz`

如果只是想快速搜索、查看头尾内容或分页阅读，通常没有必要先把 `.gz` 文件解压到磁盘。

## 核心思路

对于压缩日志，可以把内容流式解压到标准输出，再接上 `grep`、`head`、`tail`、`less` 这些常用命令。

## 常用速查

### 普通日志文件

```bash
grep '关键字' sys-info.log
head -n 100 sys-info.log
tail -n 100 sys-info.log
less sys-info.log
```

### `.gz` 压缩日志

```bash
zgrep -a '关键字' sys-info.2026-04-07.0.log.gz
zcat sys-info.2026-04-07.0.log.gz | head -n 100
zcat sys-info.2026-04-07.0.log.gz | tail -n 100
zless sys-info.2026-04-07.0.log.gz
```

### 更通用的写法

```bash
gzip -dc sys-info.2026-04-07.0.log.gz | head -n 100
gzip -dc sys-info.2026-04-07.0.log.gz | tail -n 100
gzip -dc sys-info.2026-04-07.0.log.gz | grep '关键字'
```

其中：

- `-d` 表示解压
- `-c` 表示输出到标准输出，不写回磁盘

## 最常用的几种操作

### 查关键词

```bash
zgrep -a '关键字' xxx.gz
gzip -dc xxx.gz | grep '关键字'
```

### 看前 100 行

```bash
head -n 100 xxx.log
zcat xxx.log.gz | head -n 100
```

### 看后 100 行

```bash
tail -n 100 xxx.log
zcat xxx.log.gz | tail -n 100
```

### 分页查看

```bash
less xxx.log
zless xxx.log.gz
```

### 过滤后分页

```bash
zgrep -a '关键字' xxx.gz | less
```

## 使用时要注意

- 查看压缩日志前几行通常比较快，因为解压到够用就会停止。
- 查看压缩日志最后几行时，通常还是要顺序解完整个压缩流，所以可能会慢。
- `zcat xxx.gz | tail -n 100` 不会生成解压文件，但对大日志仍然有性能成本。
- 普通 `gzip` 不适合随机跳到文件尾部直接读取，这是压缩格式本身决定的。

## 兼容压缩与未压缩日志的函数

如果你经常查日志，比较适合在 shell 里准备一组统一函数。

```bash
loghead() {
  file="$1"
  n="${2:-100}"
  case "$file" in
    *.gz) gzip -dc "$file" | head -n "$n" ;;
    *)    head -n "$n" "$file" ;;
  esac
}

logtail() {
  file="$1"
  n="${2:-100}"
  case "$file" in
    *.gz) gzip -dc "$file" | tail -n "$n" ;;
    *)    tail -n "$n" "$file" ;;
  esac
}

loggrep() {
  file="$1"
  pattern="$2"
  case "$file" in
    *.gz) gzip -dc "$file" | grep -a --color=always -- "$pattern" ;;
    *)    grep -a --color=always -- "$pattern" "$file" ;;
  esac
}

logctx() {
  file="$1"
  pattern="$2"
  lines="${3:-5}"
  case "$file" in
    *.gz) gzip -dc "$file" | grep -a -n -C "$lines" --color=always -- "$pattern" ;;
    *)    grep -a -n -C "$lines" --color=always -- "$pattern" "$file" ;;
  esac
}
```

建议放到 `~/.zshrc` 或 `~/.bashrc` 中。

常用示例：

```bash
loghead sys-info.2026-04-07.0.log.gz 100
logtail sys-info.2026-04-07.0.log.gz 100
loggrep sys-info.2026-04-07.0.log.gz 'Exception'
logctx sys-info.2026-04-07.0.log.gz 'timeout' 10
```

## 补充

如果机器上安装了 `ripgrep`，普通文本日志优先用 `rg` 往往更顺手：

```bash
rg '关键字' xxx.log
gzip -dc xxx.gz | rg '关键字'
```
