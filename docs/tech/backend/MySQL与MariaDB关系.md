## 一句话关系

> MariaDB 是从 MySQL 分叉出来并持续演进的开源数据库，在常见开发场景中与 MySQL 高度兼容。

## 背景

- MySQL 原作者是 Michael Widenius。
- Oracle 收购 MySQL 后，原作者发起了 MariaDB 项目。
- MariaDB 的目标是保持开放、社区驱动的发展路线。

## 核心关系

- 常见 SQL（建库建表、增删改查、事务、索引）在两者间迁移成本较低。
- 大多数客户端和驱动在两者间都可使用。
- 但在版本特性、默认配置、部分函数行为和运维细节上，二者仍有差异。

## 为什么学习阶段可以先用 MariaDB

- 完全开源，社区版本获取和使用都比较友好。
- Linux 发行版安装方便，适合快速搭建练习环境。
- 足够覆盖数据库基础学习主线。

## 会不会影响你学习 MySQL

> 不会影响数据库基础学习，但要保留“版本差异意识”。

你在 MariaDB 中学到的以下能力，可以直接迁移到 MySQL：

- 建库和建表
- 常见 SQL（增删改查）
- 索引和事务基础

## 一句话记忆

> MySQL 是 Oracle 维护的官方版本，MariaDB 是原作者主导的开源分支，二者基础能力高度相近，但细节并非完全一致。

## 结论

> MariaDB 可以作为 MySQL 学习与开发的可靠起点；如果要在生产环境互相迁移，请先做版本差异验证。
