## 请求方向：前端如何传参给后端

常见的请求体格式有三种：

| 格式 | Content-Type | 适用场景 |
| --- | --- | --- |
| JSON | `application/json` | 结构化数据、嵌套对象、数组 |
| 表单 | `application/x-www-form-urlencoded` | 扁平键值对 |
| FormData | `multipart/form-data` | 文件上传，或文件 + 文本混合 |

### 什么时候用 JSON

当参数里有嵌套对象、数组、Map、List 等复杂结构时，应优先使用 JSON。

```java
@PostMapping("/update")
public Boolean update(@RequestBody UserUpdateDTO dto) {
    return true;
}
```

```ts
request({
  url: '/api/user/update',
  method: 'post',
  data: payload
})
```

### 什么时候可以用表单

当参数全是扁平基础字段（如 `String`、`Long`、`Integer`）时，可以使用表单格式。

```java
@PostMapping("/query")
public PageResult query(UserQueryDTO dto) {
    return PageResult.empty();
}
```

```ts
request({
  url: '/api/user/query',
  method: 'post',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  data: new URLSearchParams(formData)
})
```

### 什么时候必须用 FormData

有文件上传时必须使用 `multipart/form-data`。

```java
@PostMapping("/upload")
public UploadResult upload(@RequestParam("file") MultipartFile file) {
    return new UploadResult();
}
```

```ts
const fd = new FormData();
fd.append('file', file);
fd.append('type', 'avatar');
request({ url: '/api/user/upload', method: 'post', data: fd });
```

## 响应方向：后端如何返回给前端

除了常规 JSON，常见还有文件流和图片字节流：

| 返回类型 | Content-Type | 前端 `responseType` | 常见用途 |
| --- | --- | --- | --- |
| JSON | `application/json` | 默认 | 列表、详情、状态 |
| 文件流（Excel/CSV） | `application/octet-stream` | `blob` 或 `arraybuffer` | 导出 |
| 图片字节流 | `image/jpeg` 等 | `blob` 或直接 `<img src>` | 图片预览 |

### JSON 返回（默认）

```java
@PostMapping("/sum")
public BigDecimal sum(@RequestBody QueryDTO dto) {
    return BigDecimal.ZERO;
}
```

前端不需要额外设置 `responseType`。

### 文件下载返回

```java
@PostMapping("/export")
public void export(HttpServletResponse response, @RequestBody QueryDTO dto) {
    // 写入文件流
}
```

```ts
request({
  url: '/api/report/export',
  method: 'post',
  data,
  responseType: 'blob'
})
```

## 实战判断指南

前端接一个新接口时，可以按这个顺序判断：

1. 看后端是否有 `MultipartFile`：有则用 `FormData`。
2. 没有文件时，看是否有 `@RequestBody`：有则传 JSON。
3. 没有 `@RequestBody` 且是扁平字段：可用表单。
4. 看后端返回是否 `void + HttpServletResponse`：是则前端用 `blob`/`arraybuffer`。

## 推荐约定

在无文件上传场景下，推荐团队统一为：

- 后端：`@RequestBody DTO`
- 前端：JSON 请求体

这样可以减少接口风格混用，降低沟通和联调成本。
